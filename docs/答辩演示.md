# 项目定位

- 面向加密资产候选发现与人工复核
- 监控 + 手动确认，不自动交易
- 目标：解释“为什么进池 / 为什么触发 / 为什么被否决”
- Web 看板 + AI 复核助手 + 回放闭环

# 课程技术点

- Prompt Engineering：结构化复核 Prompt 与 JSON 输出
- AI Agent：Evidence -> Freshness -> Fusion -> Router 多步决策
- MCP / 工具集成：Binance / OKX / market structure 适配层
- RAG 式上下文增强：候选证据 + 风险护栏 + 人工清单
- 安全护栏：阻断自动交易与提示词注入
- TDD-for-AI：Vitest + pytest 可执行需求规约

# 架构图

```text
Source Adapters
  -> Evidence
  -> Freshness Engine
  -> Fusion Engine
  -> Lifecycle Classifier
  -> Regime Classifier
  -> Lana / Skanda Router
  -> Dashboard + Manual Review
  -> Replay + Calibration
```

# v3 核心升级

- 同家族去重：避免同一事件重复加分
- 跨家族共振：保留结构、链上、榜单、舆情的独立信息
- 三态新鲜度：fresh / degrading / stale
- 显式评分：ScoreBreakdown 不做黑箱
- 人工复核：落盘、回放、离线校准

# 双模路由

- Lana = trend_long
- Skanda = exhaustion_short
- 无 market_structure.confirmation，最高只能 watchlist
- social/rank/onchain 共振不能替代结构确认
- stale / unavailable / unresolved rate-limit 阻断高置信路由

# AI 复核助手

- 接口：POST /api/ai/analyze
- 优先本地 Ollama：RESEARCH_ENABLE_LLM=1
- 未启用或失败时降级 deterministic-policy
- 输出：summary / tradePlan / checklist / evidenceUsed / invalidationRules
- 永远 manual-only，不下单

# 安全设计

- 识别自动交易请求：下单、开仓、全仓、梭哈
- 识别提示词注入：忽略规则、泄露 system prompt
- market_structure veto 优先于 social/rank/onchain
- manual feedback 不回写当前路由

# Web Demo

- 输入：分析 ORDI
- 查看候选卡：生命周期、环境、路由、最终分
- 查看执行卡：是否观察、偏多、偏空
- 查看 AI 助手：生成人工复核框架
- 查看证据页：按家族展示 freshness 和 primary evidence
- 保存人工复核：回放可见

# TDD-for-AI

- pytest 用例数量：12
- 覆盖：freshness、fusion、router、安全、AI 助手
- requirements_test.py 是可执行需求规约

# 测试示例

- discovery 强但无结构确认 -> watchlist

- auto execution -> manual-only risk flag
